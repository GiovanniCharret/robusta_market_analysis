Botão de download do Excel

  • Disponível em ambas as telas: canto superior direito no dashboard, mesma linha do breadcrumb na página do ticker.
  • Implementação: o cron passa a rodar python -m robusta run --export-xlsx ~/robusta/var/latest.xlsx a cada execução, gerando o XLSX ao lado do latest.json. nginx serve /data/latest.xlsx. Botão é um <a download> puro, sem JS.
  • O Excel contém o merged_results completo (todos os tickers do universo + todas as colunas técnica + fundamentalista), mesmo conteúdo que a flag --export-xlsx já produz hoje.
  • Nome do download: robusta-latest.xlsx.

O que mudou na revisão anterior (carteira)

  • Bloco "Sua carteira" aparece embaixo da régua do ticker buscado, como tabela compacta.
  • Cada linha mostra os mesmos indicadores do ticker buscado, alinhados em colunas: score, sinal, posição setorial, mini-régua S/R, preço.
  • A mini-régua usa as mesmas cores da régua grande (verde-oliva = suportes, marrom = resistências, círculo clay = preço) — só sem labels numéricos. Bate o olho e vê "o preço tá colado na resistência?" ou "tem folga até o próximo suporte?".
  • A linha do ticker que você está vendo (PRIO3 neste mockup) fica destacada (fundo levemente clay, ticker em clay).
  • Se o ticker buscado está na carteira, aparece um pin discreto "● na carteira" ao lado do nome no header.
  • Cliques nas linhas da carteira navegam pra ticker.html?ticker=XXXX — a mesma página, recarregada com outro ticker. Você não precisa voltar pro dashboard pra trocar.

Como a carteira é alimentada: um arquivo JSON manual em ~/robusta/var/carteira.json que o frontend lê via fetch('/carteira.json'). Você edita à mão quando comprar/vender. Formato: 

{
  "tickers": ["LREN3", "ASAI3", "GOAU4", "PRIO3", "KBLN11", "POMO4"]
}

(Padronizei pra array simples em vez do dict {"1":"lren3"...} — mais limpo no JS, sem chaves numéricas sem semântica. Mas se preferir manter o formato original, é só ajustar o leitor.)
