02

Fluxo de dados

Duas fontes alimentam dois blocos de análise; o pipeline funde tudo num RunResult, que vira JSON em disco. A API só lê esses arquivos — nunca recalcula. A linha tracejada é o handoff precos_por_ticker: a técnica entrega os preços que a fundamentalista consome.

Sólido = caminho principal de dados. Tracejado clay = handoff Dict[str, float] ticker→preço entre técnica e fundamentalista.

Contrato de saída

As duas peças mais fáceis de errar: a estrutura única que o pipeline devolve (RunResult) e o encoder que a transforma em JSON sem vazar tipos do pandas/numpy.

pipeline.py — estrutura devolvida pela Fase 5
from dataclasses import dataclass

# campo a campo igual ao JSON do contrato
@dataclass
class RunResult:
    schema_version:     int
    run_id:             str
    generated_at:       str          # ISO 8601 UTC
    robusta_version:    str          # fonte única: config
    input_universe:     list
    summary:            dict         # inclui std_vol
    technical_results:  list
    fundamental_results: list
    merged_results:     list
    portfolio_signals:  dict         # top 5 long / short
    warnings:           list
    failed_tickers:     list         # {ticker, reason}
persistence.py — encoder JSON (Fase 6)
import json, math
import numpy as np, pandas as pd

class RobustaEncoder(json.JSONEncoder):
    def default(self, o):
        if isinstance(o, (np.integer,)):
            return int(o)
        if isinstance(o, (np.floating,)):
            return float(o)
        if isinstance(o, pd.Timestamp):
            return o.isoformat()
        return super().default(o)
